# Webhook Delivery Issues

## Status change did not send webhook

Verify all are true:

- The configured trigger field in `Outbound_Trigger_Field_Name__c` changed
- The new value is included in `Outbound_Trigger_Field_Values__c`
- `Outbound_Trigger_Enabled__c` is checked (or left blank)
- A `PublicApi_Integration_Config__c` record is being supplied to the outbound event
- Linked config has `Outbound_Webhook_URL__c`
- If the source object is not `PublicApi_Submission__c`, a record-triggered Flow is calling `PublicApiOutboundStatusAction`

## Webhook record shows Failed

Check:

- Endpoint availability
- TLS/certificate validity
- Response status and response body in `PublicApi_Webhook_Delivery__c`
- If the error is a remote-site/configuration failure, use **Provision Webhook Domain** in the Request Builder and confirm the config shows `Provisioned`.

## Header problems

Review:

- `Outbound_Auth_Mode__c`
- `Outbound_Header_Name__c`
- `Outbound_Header_Value__c`
- `Outbound_Additional_Headers_JSON__c`

Auth/managed headers are reserved and may override duplicates.
