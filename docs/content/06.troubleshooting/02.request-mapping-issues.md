# Request Mapping Issues

## Mapping not applied

Check:

- `Inbound_Payload_Configuration__c` contains valid JSON
- Value references use `$<ObjectApiName>.<FieldApiName>`
- Object/field references match `Target_Object_API_Name__c`

## Validation error when saving mappings

Cause: invalid prefix or bad field API name.

Valid:

`$PublicApi_Submission__c.Status__c`

Invalid:

`$submission.Email__c`

## Field value not written

Potential causes:

- Incoming key does not match `externalKey`
- Source payload key is nested but mapping key is flat
- Field-level constraints reject value (length, picklist, type)

## Related record configuration errors

Common causes:

- selected lookup field is not a real lookup/master-detail field
- selected related object does not match the lookup target
- match field is not queryable/filterable
- `createMappings` uses a bad field API name
- more than one related record matched while `onMultipleMatches = Error`
- required fields for related-record create cannot be satisfied

Behavior:

- the API returns a specific `400` error immediately
- the target record is not inserted
- no separate related-record audit object is created in Version 1
