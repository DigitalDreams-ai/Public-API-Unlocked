# Request Mapping Issues

## Mapping not applied

Check:

- `Outbound_Payload_Configuration__c` contains valid JSON
- Value references use `$<ObjectApiName>.<FieldApiName>`
- Object/field references match configured object API names:
  - `Inbound_Target_Object_API_Name__c`
  - `Outbound_Source_Object_API_Name__c`

## Validation error when saving mappings

Cause: invalid prefix or bad field API name.

Valid:

`$PublicApi_Submission__c.Status__c`

Invalid:

`$submission.Email__c`

## Field value not written

Potential causes:

- Incoming key does not match `externalKey`
- Source payload key is nested but mapping key is flat
- Field-level constraints reject value (length, picklist, type)
