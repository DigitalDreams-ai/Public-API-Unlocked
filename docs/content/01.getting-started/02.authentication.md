# Authentication

Public API requests require:

1. `X-Api-Key` header (required)
2. Optional `x-publicapi-signature` header when inbound signature verification is enabled

No OAuth token is used for this endpoint.

## Required Header

```http
X-Api-Key: <api_key>
```

The key must match exactly one active `PublicApi_Integration_Config__c` record.

## Optional Signature Header

```http
x-publicapi-signature: <hmac_sha256_hex_of_raw_body>
```

Signature is enforced only when `Webhook_Secret__c` is populated on the matched config record.

## Failure Responses

- `401` Invalid/missing API key
- `401` Invalid signature
- `503` Multiple active config records match the same API key
