# Quick Start

## 1. Configure Integration Record

Create an active `PublicApi_Integration_Config__c` record and set:

- `Is_Active__c = true`
- `API_Key__c = <your key>`
- Optional `Webhook_Secret__c` (for inbound signature validation)
- Optional outbound fields (`Outbound_Webhook_URL__c`, auth/header/payload settings)

## 2. Assign Guest Access

Assign `PublicApi Intake API (Guest)` to the Experience Cloud site guest profile.

## 3. Send Test Request

```bash
curl -X POST "https://<site>/services/apexrest/v1/publicapi/submissions" \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: <api_key>" \
  -d '{
    "first_name": "Jane",
    "last_name": "Smith",
    "email": "jane@example.com",
    "phone": "5551234567"
  }'
```

Expected success:

```json
{
  "recordId": "a1X...",
  "message": "Submission created successfully"
}
```

## 4. Optional Postman

Use assets in [`docs/postman`](../../postman):

- `PublicApi Intake API Environment (Dev Scratch Org).postman_environment.json`
