# Outbound Delivery Configuration

Configured on `PublicApi_Integration_Config__c`.

## Core Fields

- `Outbound_Webhook_URL__c`
- `Outbound_Webhook_Secret__c`
- `Outbound_Payload_Configuration__c`
- `Outbound_Additional_Headers_JSON__c`
- `Outbound_Auth_Mode__c`
- `Outbound_Header_Name__c`
- `Outbound_Header_Value__c`
- `Outbound_Content_Type__c`

## Remote Site Setting Behavior

- Remote Site Settings are provisioned during outbound delivery execution, not when saving config records.
- Provisioning is domain-scoped (`https://partner.example.com/path` provisions `https://partner.example.com`).
- If provisioning fails, the delivery is recorded as `Failed` with error details in the delivery record.

## Auth Modes

- `None`
- `ApiKeyHeader`
- `BearerToken`
- `BasicAuth`
- `HmacSignature`

If auth mode is blank, no auth header is added.

## Delivery Error Codes

When a delivery fails, `PublicApi_Webhook_Delivery__c.Error_Code__c` is populated with:

- `MAPPING_ERROR`
- `AUTH_HEADER_ERROR`
- `REMOTE_SITE_ERROR`
- `WEBHOOK_RESPONSE_ERROR`
- `WEBHOOK_CALLOUT_ERROR`
- `UNKNOWN_ERROR`
