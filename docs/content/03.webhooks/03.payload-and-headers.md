# Payload and Headers

## Default Outbound Payload

If you do not override mappings, outbound payload includes:

```json
{
  "name": "Jane",
  "last_name": "Smith",
  "email": "jane@example.com",
  "phone": "5551234567",
  "gclid": "test-gclid",
  "createdDate": "2026-03-03T05:38:21.000Z"
}
```

Actual payload is controlled by `Outbound_Payload_Configuration__c`.

## Standard Outbound Headers

- `Content-Type` (defaults to `application/json`)
- `X-PublicApi-Event-Type: status_changed`

Additional auth/custom headers are applied from configuration.

## Additional Headers JSON

Supported formats:

Object format:

```json
{
  "X-Partner-Id": "abc123",
  "X-Source": "public-api"
}
```

Array format:

```json
[
  { "name": "X-Partner-Id", "value": "abc123" },
  { "name": "X-Source", "value": "public-api" }
]
```
