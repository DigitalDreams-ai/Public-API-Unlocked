# Webhooks Overview

Outbound webhook delivery runs when:

1. A submission status changes
2. New status is one of:
   - `Referred Out`
   - `Converted`
3. A `PublicApi_Intake_Tracking__c` link exists for the submission
4. Matched config has `Outbound_Webhook_URL__c`

Delivery is asynchronous via `PublicApiStatusUpdateQueueable`.

## Remote Site Provisioning

- No config trigger is used for remote-site creation.
- On outbound delivery execution, Salesforce provisions a domain-scoped Remote Site Setting for `Outbound_Webhook_URL__c` when a valid session is available.
- Provisioning runs at most once per webhook domain per transaction.

## Delivery Logging

Each attempt writes `PublicApi_Webhook_Delivery__c` and updates latest-delivery summary fields on `PublicApi_Integration_Config__c`.
