# Webhooks Overview

Outbound webhook delivery runs when:

1. A source record status changes
2. Matched config has `Outbound_Trigger_Enabled__c = true` (or blank, which defaults to true)
3. The configured outbound trigger field changes to one of the configured trigger values:
   - `Outbound_Trigger_Field_Name__c`
   - `Outbound_Trigger_Field_Values__c`
4. A `PublicApi_Integration_Config__c` record is supplied
5. Matched config has `Outbound_Webhook_URL__c`

Delivery is asynchronous via `PublicApiStatusUpdateQueueable`.

## Trigger Entry Points

- `PublicApi_Submission__c` includes a built-in trigger for status changes.
- Any other outbound source object is supported through the invocable Apex action:
  - `PublicApiOutboundStatusAction`
- For objects other than `PublicApi_Submission__c`, use a record-triggered Flow to:
  - detect the relevant status change
  - determine the target `PublicApi_Integration_Config__c`
  - invoke `PublicApiOutboundStatusAction` with record id, config id, previous status, current status, and optional changed-at timestamp

## Remote Site Provisioning

- No config trigger is used for remote-site creation.
- Admins provision the webhook domain explicitly from the Request Builder.
- Provisioning writes status back to the config record using:
  - `Outbound_Remote_Site_Status__c`
  - `Outbound_Remote_Site_Domain__c`
  - `Outbound_Remote_Site_Name__c`
  - `Outbound_Remote_Site_Last_Error__c`
- Outbound delivery does not attempt provisioning on the fly.
- If the current webhook domain is not provisioned, delivery fails fast and is logged as a configuration error.

## Delivery Logging

Each attempt writes `PublicApi_Webhook_Delivery__c` and updates latest-delivery summary fields on `PublicApi_Integration_Config__c`.

- If the source object is `PublicApi_Submission__c`, the delivery record also populates `Submission__c` on `PublicApi_Webhook_Delivery__c`.
- For any other source object, the delivery record stores:
  - `Source_Object_API_Name__c`
  - `Source_Record_Id__c`
