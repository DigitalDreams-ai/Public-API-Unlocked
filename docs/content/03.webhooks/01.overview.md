# Webhooks Overview

Outbound webhook delivery runs when:

1. A `PublicApi_Submission__c` record changes
2. Matched config has `Outbound_Trigger_Enabled__c = true` (or blank, which defaults to true)
3. The configured `PublicApi_Submission__c` trigger field changes to one of the configured trigger values:
   - `Outbound_Trigger_Field_Name__c`
   - `Outbound_Trigger_Field_Values__c`
4. The submission already links to one `PublicApi_Integration_Config__c` record
5. Matched config has `Outbound_Webhook_URL__c`

Delivery is asynchronous through the outbound platform-event pipeline:

- `PublicApiSubmissionStatusUpdateTrigger`
- `PublicApi_Outbound_Request__e`
- `PublicApiOutboundRequestTrigger`
- `PublicApiOutboundRequestQueueable`

## Trigger Entry Points

- `PublicApi_Submission__c` includes the only built-in outbound trigger.
- Outbound is not routed from arbitrary source objects in the package core.
- Trigger detection is always evaluated on `PublicApi_Submission__c`.
- The default built-in trigger is `Processing_Status__c = Succeeded`.
- Payload mappings can still target the configured business record because delivery resolves the payload source from:
  - `PublicApi_Submission__c.Target_Object_API_Name__c`
  - `PublicApi_Submission__c.Target_Record_Id__c`

## Remote Site Provisioning

- No config trigger is used for remote-site creation.
- Admins provision the webhook domain explicitly from the Request Builder.
- Provisioning writes status back to the config record using:
  - `Outbound_Remote_Site_Status__c`
  - `Outbound_Remote_Site_Domain__c`
  - `Outbound_Remote_Site_Name__c`
  - `Outbound_Remote_Site_Last_Error__c`
- Outbound delivery does not attempt provisioning on the fly.
- If the current webhook domain is not provisioned, delivery fails fast and is logged as a configuration error.

## Delivery Logging

Each attempt writes `PublicApi_Webhook_Delivery__c` and updates latest-delivery summary fields on `PublicApi_Integration_Config__c`.

- Delivery records use `PublicApi_Submission__c` as the source and also populate `Submission__c` on `PublicApi_Webhook_Delivery__c`.
