# Module Structure

Public API code is organized by domain to keep responsibilities explicit:

- `classes/publicApi/api`: REST resources and request parsing.
- `classes/publicApi/submission`: submission creation workflow and domain exceptions.
- `classes/publicApi/config`: config repository/service, remote-site management, config trigger handler.
- `classes/publicApi/mapping`: runtime mapping parser/validator/application.
- `classes/publicApi/webhook`: outbound delivery orchestration, queueing, header/auth handling, delivery logging.
- `classes/publicApi/ui`: payload/header builder Apex controller and template seeding helpers.
- `classes/publicApi/testSupport`: smoke test and shared verification-style tests.

LWC components remain in `lwc/`:

- `publicApiJsonPayloadBuilder`
- `publicApiJsonPayloadBuilderNode`
- `publicApiPrettyJson`

## Design Intent

- API endpoints stay thin and delegate.
- Mapping runtime logic is isolated from builder-template concerns.
- Queueables orchestrate only; delivery mechanics are in dedicated services.
- Config retrieval is centralized through one repository path.
