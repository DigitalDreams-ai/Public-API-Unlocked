# Submissions Overview

`PublicApiSubmissionApi` exposes a single inbound endpoint:

```http
POST /services/apexrest/v1/publicapi/submissions
```

## Supported Content Types

- `application/json`
- `application/x-www-form-urlencoded`

## Mapping Behavior

Inbound payload keys map to fields on the configured target object using `Inbound_Payload_Configuration__c` on the matched integration config.

Canonical field reference format:

`$<ObjectApiName>.<FieldApiName>`

Examples:

- `$PublicApi_Submission__c.Status__c`
- `$PublicApi_Submission__c.Received_At__c`
- `$PublicApi_Submission__c.CreatedDate`

## Related Record Behavior

If `Related_Record_Configuration__c` is configured, inbound processing can:

- match one existing related record
- create one related record if no match is found and `createIfMissing = true`
- set one lookup field on the target record before insert

Version 1 behavior is intentionally narrow:

- only one related-record rule is supported
- matched related records are not updated
- bad related-record config fails the request immediately
- no separate inbound audit object is created just for related-record errors

## Public API Submission Audit Record

Inbound processing always creates or updates a `PublicApi_Submission__c` audit record.

When `Target_Object_API_Name__c = PublicApi_Submission__c`, the audit record is also the target record.

The audit record captures:

- `PublicApi_Integration_Config__c`
- `Created_By_PublicApi_API__c = true`
- `Received_At__c = System.now()`
- `Raw_Payload__c` (truncated to 131072 chars)
- `Request_Content_Type__c`
- `Target_Object_API_Name__c`
- `Target_Record_Id__c`
- `Processing_Status__c`
- `Related_Record_Status__c`
- `Processing_Error_Category__c`
- `Processing_Error_Message__c`
- `Effective_Config_JSON__c`
